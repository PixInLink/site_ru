## 5.1. Оценка рынка

### Общий объём и динамика

Российский рынок ПО в 2025 году вырос на 23% и достиг **1 491,8 млрд рублей**. Рынок веб-разработки и CMS оценивается в ~15 млрд ₽ с ростом 8–12% в год (данные из условия задачи). Интернет-торговля в РФ в 2025 году составила **11,5 трлн рублей** (+28% год к году), что создаёт структурный спрос на сайты и онлайн-витрины.[^1][^2]

В 2025 году в России насчитывалось **6,7 млн субъектов МСП** с рекордными темпами роста оборота (+6% = +1,8 трлн рублей). По данным Soware, около **40% МСП нуждаются в сайте**, но значительная часть ещё не имеет полноценного веб-присутствия. Таким образом, адресуемая аудитория первичного рынка составляет ~2,7 млн компаний — из которых реально активна и платежеспособна, по оценке, 15–20% (~400–550 тыс. компаний).[^3][^4]

### Готовность SMB платить за цифровые решения

66% компаний МСБ увеличили расходы на digital в 2025 году, и 63% прогнозируют дальнейший рост в 2026 году. При этом главный барьер цифровизации для 65% компаний — **высокая стоимость внедрения**: именно здесь GitHub CMS с TCO $40–100/год имеет принципиальное ценовое преимущество. Треть МСП в 2025 году показала рост оборотов и делает ставку на ИИ-инструменты, что коррелирует с позиционированием продукта.[^5][^6][^3]

### Позиция WordPress и Tilda

По данным W3Techs, на начало 2026 года WordPress обслуживает **около 43% всех сайтов в мире** и сохраняет заметную долю в российском сегменте. Tilda захватила **60,3% рынка e-commerce-конструкторов** в России, резко нарастив долю среди всех CMS с 2,75% в 2023 до 18,51% в 2024 году — во многом благодаря уходу западных конкурентов после санкций. Это создаёт уязвимость: Tilda монополизирует «простой» сегмент, но её SaaS-зависимость и отсутствие GEO/AI-возможностей открывают нишу.[^7][^8]

Средняя стоимость разработки сайта под ключ для малого бизнеса в 2025–2026 годах: **30 000–100 000 ₽** единовременно + 6 000–18 000 ₽/год на поддержку. На этом фоне GitHub CMS с шаблонами и промптами позволяет SMB сделать сайт самостоятельно за значительно меньшую сумму — это не просто экономия, а смена категории.[^9][^10]

## 5.2. УТП и потенциал

### AI-видимость — сильный, но «сырой» для SMB аргумент

Структурированные данные (JSON-LD) дают **до 40% больше цитирований в AI-поисковиках**. Контент с правильной разметкой Schema.org имеет **в 2,5 раза больше шансов** попасть в AI-ответы. Согласно данным из условия задачи, статические сайты с JSON-LD цитируются на 30–45% чаще (SearchBridge AI, 2026). При этом 47% B2B-покупателей начинают поиск в AI-инструментах (Gartner, 2025).[^11][^12]

**Проблема:** среднестатистический владелец ИП или ООО не знает термина «GEO» и не понимает разницы между JSON-LD и плагином Yoast. УТП необходимо **переформулировать в язык боли**:

> ❌ «AI-видимость из коробки с 22 типами Schema.org»  
> ✅ «Ваш сайт будет появляться в ответах ChatGPT и Яндекс AI — автоматически, без доплат»

### Что мы решаем лучше конкурентов

| Боль SMB | GitHub CMS | WordPress | Tilda |
|----------|-----------|-----------|-------|
| Высокая стоимость в год | ✅ $40–100/год | ❌ $300–1200/год | ❌ $150–240/год (SaaS) |
| Медленная загрузка (плохой SEO) | ✅ TTFB ≤200ms | ❌ 600–1200ms | ⚠️ Зависит от блоков |
| Уязвимости и взломы | ✅ Нет БД = нет SQL-инъекций | ❌ 96% взломанных сайтов — WordPress[^13] | ✅ SaaS защищает |
| Нет GEO/AI-видимости | ✅ 22 типа Schema.org | ⚠️ Плагины Yoast/RankMath (платно) | ❌ Отсутствует |
| Зависимость от SaaS | ✅ Контент в Git, независимость | ✅ Self-hosted | ❌ Полная зависимость |
| «Я не программист» | ⚠️ Node.js + Git порог | ❌ Плагины = хаос | ✅ Drag&Drop |

### Порог входа — критическая слепая зона

Установка Node.js + Git + VS Code — это **3–4 часа для неподготовленного пользователя** и реальный барьер для российского SMB. Согласно исследованиям Soware, **65% МСП называют высокий порог входа и нехватку компетенций** основными барьерами цифровизации. Система промпт-шаблонов частично решает проблему после установки, но не до неё.[^3]

**Совет предлагает два пути снижения барьера:**
1. **Онбординг-видео** (YouTube/Rutube): «Сайт для бизнеса за 90 минут» — с GitHub CMS шаг за шагом
2. **Партнёрский канал**: разработчики-фрилансеры делают установку и настройку за 5 000–15 000 ₽ (первая конверсия для фрилансера), клиент получает готовый сайт + автономность

### Система промпт-шаблонов — реальное конкурентное преимущество

Промпт-шаблоны — **не «костыль», а ключевой дифференциатор**. Статические генераторы (Hugo, Astro, Eleventy) не имеют ничего подобного. Это прямой мостик между «я не умею программировать» и «у меня есть профессиональный сайт». 45+ файлов промптов, покрывающих 15 типов страниц магазина, 11 вариантов раздела «О компании» — это серьёзный производственный актив, который сложно воспроизвести быстро.[^14]

## 5.3. Монетизация и ценовая стратегия

### Анализ трёх вариантов (ToT)

**Вариант А: Open-source + маркетплейс тем**

*Плюсы:* Максимальный охват, инструмент для сарафанного радио, разработчики как соавторы тем.  
*Минусы:* Конверсия в покупку тем у стартапов без имени — 0,5–2%. При 1 000 пользователей = 5–20 продаж. Без масштаба не работает.  
*Вердикт:* Преждевременно для v0.1.4. Подходит как долгосрочная цель при 10 000+ пользователях.

**Вариант Б: Платная базовая версия ($10 однократно)**

*Плюсы:* Немедленный денежный поток, низкий барьер ($10 ≪ $70–200/год Tilda), один платёж психологически проще.  
*Минусы:* Низкий LTV, нет рекуррентного дохода, сложнее обосновать ценность незнакомому пользователю.  
*Вердикт:* **Оптимален как стартовая модель для РФ** в связке с маркетплейсом тем.

**Вариант В: Freemium (3 страницы бесплатно, полная — $10)**

*Плюсы:* Классическая SaaS-воронка, пользователь «пробует перед покупкой», лучший инструмент для конверсии.  
*Минусы:* Технически сложнее реализовать ограничения в статической архитектуре. Риск, что 3 страниц «хватит» для многих клиентов.  
*Вердикт:* Привлекателен, но требует разработки механизма ограничений — дополнительная сложность.

### Рекомендация Совета: Гибридная модель «Б + маркетплейс»

```
Этап 1 (0–6 мес.): Freemium с широкой базой
  └── Базовый шаблон: БЕСПЛАТНО (GitHub open-source)
  └── Полный пакет (45 промптов + 3 темы): 990 ₽ / $10 единовременно
  └── Позиционирование: «Дешевле одного месяца Тильды»

Этап 2 (6–18 мес.): Маркетплейс тем
  └── Премиум-темы: 490–1 490 ₽ ($5–15)
  └── Тематические пакеты (ресторан, салон, юрист): 990–2 490 ₽
  └── Реферальная программа для фрилансеров: 30% комиссия

Этап 3 (18+ мес.): SaaS-опции
  └── GitHub CMS Cloud: управляемый хостинг 499 ₽/мес
  └── AI-генерация контента внутри платформы (API ChatGPT/Claude)
```

### Ценовое обоснование для РФ

Ценообразование должно учитывать рублёвый рынок. **990 ₽** — психологически ниже «1 000 рублей», воспринимается как «почти бесплатно» на фоне 6 000–18 000 ₽/год за Tilda. Стоимость тем в маркетплейсе: **490–1 490 ₽** ($5–15 по текущему курсу) — соответствует ценам российских тем для WordPress и Craftum.[^8][^15]

## 5.4. Стратегия выхода на рынок

### Фаза 1: Органический запуск (месяцы 1–3)

**Приоритетные каналы для РФ:**

1. **Habr.com** — аудитория разработчиков и tech-savvy менеджеров. Статья «Как я сделал CMS на Vue 3 + Vite SSG с AI-видимостью» — это не реклама, а нативный контент. Средний охват технической статьи на Хабре: 5 000–30 000 просмотров.[^16]
2. **VC.ru** — аудитория предпринимателей. Статья в формате «Кейс: сайт для малого бизнеса за $40/год вместо Tilda». Сильный SEO-след в русскоязычном сегменте.
3. **Telegram-каналы** по веб-разработке, SEO, маркетингу для МСП — платные посевы 5 000–15 000 ₽ за размещение, аудитория уже прогрета.
4. **GitHub** — хорошее README + демо-сайт = органический трафик от разработчиков. Jamstack Platform Market на 54% состоит из SME-пользователей.[^17]

**Product Hunt** — актуален для получения международного трафика и бэклинков, но конверсия в платящих российских пользователей будет низкой. Рекомендуется как PR-ход на этапе 2–3.

### Фаза 2: Фрилансерский канал (месяцы 3–9)

Рынок фриланса в РФ активен: более 12 млн самозанятых к концу 2024 года. Разработчики-фрилансеры — **идеальный канал**: они делают 5–20 сайтов в год, ищут быстрые решения, могут «упаковать» GitHub CMS как услугу для клиентов.[^18]

Предложение для фрилансеров:
- Реселлерская программа: покупают пакет за 1 990 ₽, перепродают клиенту за 5 000–15 000 ₽ (установка + настройка)
- Комиссия 30% с продаж тем, которые привёл реферал
- Готовые коммерческие предложения / КП для их клиентов в формате PDF

Средний чек разработки простого сайта у фрилансера: **20 000–300 000 ₽**. GitHub CMS снижает время разработки в 5–10 раз → фрилансер зарабатывает больше при той же цене для клиента.[^19]

### Критическая масса и самоокупаемость

- **Точка безубыточности:** ~80 продаж полного пакета по 990 ₽ = ~80 000 ₽/год (покрывает базовые расходы)
- **Устойчивый рост:** 300+ активных пользователей создают органическое «сарафанное радио», 50+ сайтов в production = портфолио для привлечения
- **Целевой показатель 12 мес.:** 2 000–5 000 установок, 200–600 платящих пользователей

### SEO/GEO vs платный трафик

Для стартового этапа — **только органика**. Yandex Direct и VK Ads требуют $500–2 000/мес бюджета для устойчивого результата при CAC $15–40 на целевое действие. Контент-маркетинг через Habr/VC + YouTube-видео «как сделать сайт с нуля за 90 минут» — затраты близки к нулю, долгосрочный SEO-след. На этапе 2 (после первых 500 пользователей) — ретаргетинг по аудитории посетителей демо-сайта.

## 5.5. Риски и слепые зоны

### Риск 1: Копирование модели (HIGH, но управляемый)

Промпт-шаблоны легко скопировать технически. Защита — **скорость** и **экосистема**: 45 шаблонов × постоянное обновление под новые типы контента, интеграция с конкретными платёжными системами РФ (Тинькофф), локализованные примеры. Hugo и Astro могут сделать похожее, но не сделают за 6–12 месяцев — у них другие приоритеты.[^20][^14]

### Риск 2: WordPress/Tilda добавят AI-генерацию (MEDIUM)

Tilda уже активно развивает AI-блоки в 2025 году и готовится к выходу на рынок публичных инвестиций. Однако **SaaS-зависимость и ценовая модель Tilda** не изменятся: добавление GEO-разметки в конструктор не уберёт $150+/год стоимость. WordPress также активно интегрирует AI, но TTFB проблема системная — архитектурная, не плагинная.[^21][^22][^23][^24]

**Ключевая защита:** GitHub CMS — это не плагин, это **архитектурная позиция**. Статические сайты принципиально быстрее — это не фича, которую можно «допилить» в WordPress плагином.[^22]

### Риск 3: GitHub меняет условия (LOW)

GitHub Pages остаётся бесплатным и стабильным с 2008 года. В апреле 2026 обнаружена и оперативно устранена уязвимость CVE-2026-3854, что говорит о зрелости платформы. Альтернативы (Netlify, Vercel, VPS на nginx) доступны без рефакторинга — Vite SSG генерирует стандартный HTML. Cloudflare Pages как fallback — настраивается за 20 минут.[^25][^26]

### Риск 4: Юридические ограничения РФ (MEDIUM, управляемый)

**54-ФЗ:** Приём платежей через кнопку-ссылку на внешнюю кассу (Тинькофф) требует наличия онлайн-кассы у продавца. Для самого GitHub CMS как продукта это не проблема — покупатель использует стороннюю платёжную систему. Для клиентов-пользователей с магазинами: нужна явная документация «что нужно для легальной торговли» + партнёрство с Тинькофф Кассой.[^27][^28][^29]

**Импортозамещение:** Указ о замене иностранного ПО в КИИ не затрагивает коммерческий SMB-сегмент. Для GitHub как хостинга: риск блокировки GitHub.com существует (аналогично Twitter/Instagram), но маловероятен — у GitHub большая российская аудитория разработчиков. Митигация: документировать миграцию на Gitea/GitLab за 1 час.[^30][^31][^32]

**Персональные данные (152-ФЗ):** Статический сайт без форм обработки персональных данных не нарушает закон. Если на сайте есть форма обратной связи через внешний сервис — ответственность на уровне того сервиса.

### Слепые зоны, выявленные скептиком (Omission Errors)

1. **Отсутствие мобильного редактора.** Контент-менеджеры SMB в РФ часто работают со смартфона. VS Code + GitHub CLI — это десктоп-инструменты. Необходима интеграция с GitHub Mobile или веб-редактором (github.dev).

2. **Нет механизма уведомлений/поддержки.** После продажи пакета клиент остаётся один на один с Node.js. Без чата поддержки (хотя бы Telegram-группа) churn будет высоким — пользователь не дойдёт до «зелёного» сайта.

3. **Российский хостинг как альтернатива VPS.** Часть SMB-клиентов не готова администрировать nginx на Ubuntu. Нужна инструкция по деплою на российские хостинги с cPanel (Timeweb, Beget) — они поддерживают статику, стоят 150–300 ₽/мес.

4. **Отсутствие русскоязычной документации.** Это первичный рынок РФ/СНГ, но README и документация на английском создают дополнительный барьер.

5. **Не монетизирован консультационный канал.** Услуга «Настройка сайта под ключ на GitHub CMS» — высокомаржинальная: 2–4 часа работы = 5 000–20 000 ₽. Это выгоднее, чем продажа шаблонов на старте.
